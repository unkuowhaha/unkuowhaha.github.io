虚拟地址、线性地址、物理地址

虚拟地址(virtual address)–段转换–>线性地址(liner address)–页转换–>物理地址(physical address)

物理页

物理页是把总物理内存以指定大小划分为页单位的概念。在实验中一页的大小是4KB。

当页面为空闲时，pp_ref=0。
空闲页列表初始化后:

NULL<–page<–page<–page…page<–page_free_list

申请新的物理页时，把page_free_list指向的物理页从空闲页列表中申请出来，alloced_page->pp_ref置1。
闲置物理页时，page->pp_ref需要为0才可闲置，从尾部插入闲置的物理页。

页转换(page translation)

页转换指把线性地址转换为相应的物理地址。

页目录(page_dir)，存储着页表(page_table)的物理地址和相应的权限。
页表(page_table)存储着物理页的地址和相应的权限。
页目录和页表都是页。

buddy system

对于宏定义define offsetof(TYPE, MEMBER) ((size_t) &((TYPE*)0)->MEMBER)的理解
对于这个宏可以大致分为5步：

  1. 0
  2. ((TYPE *)0)
  3. ( ((TYPE *)0)->MEMBER )
  4. &( ((TYPE *)0)->MEMBER )
  5. ( (size_t) &( ((TYPE *)0)->MEMBER )

1、内存地址开始于0；
2、将0转换为type类型的结构体指针，换句话说就是让编译器认为这个结构体是开始于程序段起始位置；
3、引用结构体中MEMBER成员；
4、取地址符&，我们这里不关注结构体成员的内容，只取该成员的地址；
5、将取到的地址强制转换为size_t类型
